## 5. 遇到的问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| npm项目名不能大写 | npm限制 | 先用小写创建，完成后重命名 |
| GitHub连接失败 | 网络问题 | 设置cron自动重试推送 |
| 照片显示大小不一 | 卡片宽度不固定 | 改为固定18rem宽度，照片改为16:9比例 |
| 预设节点内容不符 | 用户有真实记录 | 支持"编辑预设节点"功能，复制为自定义后修改 |
| 首次打疫苗标题不准确 | 用户想表达真实情况 | 在管理后台编辑预设节点功能 |

---

## 6. 解决方案

| 场景 | 方案 |
|------|------|
| 数据更新后用户看不到新数据 | 静态生成，数据编译进HTML，构建后自动更新 |
| 添加新记录 | localStorage临时存储 + 导出代码功能 |
| 编辑预设节点 | 复制为自定义记录 → 编辑 → 保存 |
| 照片管理 | 用户用PicGo上传GitHub，粘贴CDN链接 |

## 7. 经验与教训

1. **静态生成优于运行时加载**
   - AndyAI数据在构建时编译进HTML，更新即部署
   - PromptWorld运行时fetch JSON，容易被缓存

2. **数据与代码分离**
   - 大量数据不应嵌入JS（PromptWorld教训）
   - 使用JSON文件或数据库

3. **localStorage适合临时存储**
   - 适合管理后台的草稿、临时记录
   - 不能依赖它作为唯一数据源

4. **预设数据需要灵活性**
   - 预设里程碑可能与实际不符
   - 设计支持"复制后修改"的编辑模式

5. **图片CDN选择**
   - jsDelivr CDN稳定可靠
   - 支持GitHub资源直链
